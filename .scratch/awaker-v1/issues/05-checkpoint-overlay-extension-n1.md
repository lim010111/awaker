# 05 — 체크포인트 오버레이 + 체류 연장 + 북극성 N1 측정

Status: ready-for-human

## What to build

제품의 심장 슬라이스. teacher 룰 양성 × dwell(초안 90s, 튜닝 변수) 통과 시 SYSTEM_ALERT_WINDOW 바텀 시트로 체크포인트를 후보 앱 위에 띄운다. 메시지는 LLM 호출 없이 **로컬 템플릿 풀 stub**(소수 문장이면 충분) + **메시지 슬롯 주입**(누적 분 버킷, 세션 N번째 연장 — CONTEXT.md)으로 구성한다. ADR-0001의 prefetch 구조는 이 단계에서 뼈대만 유지(신뢰도 곡선 미검증 — ADR-0011).

선택지는 둘: **체류 연장** → 시트 닫힘 + 세션 연장 카운터 증가 + 다음 표시 높이 증가(1회차 25% → 최대 70% cap, ADR-0003 점진적 가림), **자발 종료** → 06으로 위임(이 슬라이스에선 버튼만 존재해도 됨). 세션 종료 시 카운터·가림 리셋.

체크포인트 표시 시각과 표시 후 1분 이내 후보 앱 이탈 여부(북극성 N1, ADR-0007)를 이벤트로 기록하고 인앱에서 집계를 확인할 수 있게 한다.

## Acceptance criteria

- [ ] 실사용 중 임계 통과 시 체크포인트 바텀 시트가 후보 앱 위에 뜬다
- [ ] 메시지에 실측 슬롯 값(누적 분 버킷, N번째 연장)이 주입돼 있다
- [ ] 연장을 반복하면 시트 높이가 자라고 70%를 넘지 않으며, 상단 30%는 항상 후보 앱이 보인다
- [ ] 세션 종료 시 가림 점유율·연장 카운터가 리셋된다
- [ ] N1 이벤트(표시 시각, 1분 내 이탈 여부)가 기록되고 인앱에서 집계를 볼 수 있다
