# 02 — 후보 앱 세션 감지: 포그라운드 서비스 + 세션 상태 머신

Status: ready-for-human

## What to build

프로토타입의 뿌리 슬라이스. 후보 앱(기본셋 최소 2~3개 하드코딩, 예: YouTube·Instagram)이 포그라운드에 뜨면 세션 시작, CONTEXT.md 세션 정의대로 *5분 이상 포그라운드를 떠나면* 세션 종료로 판정하는 상태 머신을 포그라운드 서비스 + UsageStatsManager 폴링으로 구현한다 (ADR-0004 권한 키트, ADR-0012 스택).

세션 시작/종료 이벤트는 로컬 DB에 기록하고, 인앱 화면에서 (a) 필수 권한(UsageStats·알림) 부여 상태와 (b) 세션 이력을 확인할 수 있어야 한다. 1차 코호트는 개발자 본인 기기뿐이므로 온보딩 폴리시는 불요 — 권한 부여 동선만 동작하면 된다.

배터리·생존성 측정이 이 슬라이스의 숨은 절반이다: 상시 상주가 성립하지 않으면 제품 전체가 무효다.

## Acceptance criteria

- [x] 후보 앱을 열면 세션 시작, 5분 룰 경과 후 세션 종료 이벤트가 기록된다 (짧은 이탈 후 복귀는 같은 세션 유지)
- [x] 인앱 화면에서 권한 부여 상태와 세션 이력을 확인할 수 있다
- [ ] 포그라운드 서비스가 본인 기기에서 24시간 상주 후 살아 있다 (OEM 킬러/Doze 생존)
- [x] 상주-대기 상태(세션 없음)의 배터리 소모 실측치가 이슈 코멘트에 기록된다
