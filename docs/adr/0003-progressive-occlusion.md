# 체크포인트 escalation은 점진적 가림으로 표현한다

E1(순수 메타인지, [[CONTEXT.md]])로 못박았지만 반복 사용자에 대해 *어떤 종류의 압력 누적*은 필요했다. 이 압력을 *선택지 제거*가 아니라 **체크포인트 바텀 시트의 높이 증가**로 표현한다. 같은 세션에서 [[Extension]]이 누적될수록 시트가 자라며(예: 25% → 35% → 50% ...), 두 선택지(체류 연장 / 자발 종료)는 *항상 살아 있음*. 시각 점유율 자체가 비언어적 메타인지 신호 — 사용자는 글자를 읽지 않아도 *몸으로* 자각.

## Considered Options

- **풀 takeover (100%까지 자람)** — 버튼이 기술적으로 살아있을 뿐 사용자 경험상 막힘 → *E1을 깨고 E3로 변질*. 비채택.
- **사용자 옵션으로 cap 상향** — 매력적이나 v1에서 검증 안 된 상태로 설정 표면을 늘리지 않음. v2에서 데이터 보고 재검토.

## Consequences

- **천장은 화면 높이의 70%.** 어떤 횟수든 위 30%는 항상 후보 앱 보임 → "압박은 하되 막진 않는다" 라는 제품 약속의 *시각적 증거*.
- **리셋 정책**: 후보 앱이 *연속 5분 이상* 포그라운드를 떠나거나, *자발 종료* 가 성공 검증되면 즉시 리셋. 매 세션이 fresh — "처벌이 누적된다"는 느낌 차단.
- 시트가 자라는 *곡선* (선형/지수/계단)은 튜닝 가능 변수로 v1 출시 후 데이터 기반 조정.
- E1로 보이지만 *시각적 escalation* 이 있는 구조 — 코드 봤을 때 모순처럼 보일 수 있어 본 ADR이 그 모순을 해소하는 명시 문서.
