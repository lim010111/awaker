# LLM 데이터 경계, 캐싱, 비용 캡 정책

체크포인트 메시지는 클라우드 LLM이 생성하지만, *사용자 행동·일정 raw 데이터*는 클라우드로 나가지 않는다. 행동 데이터를 외부에 누적시키는 건 본 제품의 신뢰 약속("당신을 *대신해서* 깨우는, *프로파일링하지 않는* 도우미")을 정면으로 깬다. 동시에 메시지 풍부함을 위해 LLM이 *질감 있는 컨텍스트*는 받아야 하므로 — **버킷팅된 추상 토큰**으로 경계를 그어 운영한다.

## LLM이 받는 것 — 버킷팅 컨텍스트

- **시간대**: `"새벽 1시~2시 사이"` (raw `"01:23:45"` 아님)
- **누적 사용**: `"30~60분"` 구간 (raw `"38분 12초"` 아님)
- **Extension 카운트**: 정수 그대로 (작은 정수, 위험 낮음)
- **후보 앱 카테고리**: `"단형 영상"`, `"무한 피드"` (앱 이름 아님)
- **자세**: `"누워있음"`, `"기울어짐"` 같은 라벨 (자이로 raw 각도 아님)

## LLM이 절대 받지 않는 것

- 정밀 타임스탬프, raw 센서 시계열, 앱 이름/콘텐츠, 사용자 식별자, 정확한 일정/캘린더 raw

## 메시지 캐싱 — 세 층 분리

1. **Prefetch 캐시** — 예비 임계 ~ 발동 임계 사이 *1회용 임시 보관*. 표시 / 임계 미달 / 세션 종료 셋 중 하나로 폐기. ADR-0001 참조.
2. **표시된 메시지** — *저장 안 함, 재사용 0*. 메타인지 보호(반복 인지 차단) + 프라이버시 보호.
3. **로컬 템플릿 풀** — 출시 빌드에 박힌 50~200개 문장 톤. LLM 실패/오프라인/비용 캡 초과 시 *silent fallback*. 사실 수치는 동일한 [[message-slot]] 방식으로 주입.

## 비용 캡

**사용자당 일 8회 LLM 호출**. 출발점이며 출시 후 실측으로 튜닝. 초과 시 그날 나머지는 로컬 템플릿으로 자동 전환, *사용자에겐 알리지 않음* — 백엔드 상태를 UX로 누출하지 않음. 시스템은 *절대 침묵하지 않음*.

## Considered Options

- **P1 raw 데이터 전송** — 메시지 임팩트 최대지만 행동/일정 프로파일링 가능. 본 제품 신뢰 약속 위반. 비채택.
- **P3 LLM 없는 순수 로컬** — 안전하지만 *AI 자산* 정체성 손실 + 반복 인지 위험. 로컬 템플릿은 *fallback*으로만 사용.
- **표시 메시지 재사용 캐싱** — 비용 절감되지만 사용자가 "어디서 본 문장" 인지 → 메타인지 마법 깸. 비채택.
- **fallback = 침묵** — 발동해야 할 순간에 침묵하면 *제품 가치 실패*. 비채택.

## 데이터 경로 (ADR-0008 반영)

```
Android (버킷팅 토큰 빌드)
  → Awaker 자체 백엔드 (TS, OpenRouter SDK 호스트)
  → OpenRouter 게이트웨이 (ADR-0008)
  → underlying provider (Anthropic / Google / ...)
```

본 ADR의 *버킷팅·재사용 0·캐시 정책*은 모든 hop에 적용. 각 hop의 *로깅·보존 정책*이 본 ADR과 호환되어야 — 백엔드 정책은 ADR-0009, OpenRouter·provider 정책은 ADR-0008 출시 전 확인 체크리스트.

## Consequences

- LLM 프롬프트 템플릿은 *버킷 토큰 시그니처*로 표준화 → 모델 교체 시 마이그레이션 단순.
- *"행동 데이터는 폰을 떠나지 않습니다"* 가 진짜 제품 약속이 되어 마케팅 자산화.
- 로컬 템플릿 풀 *제작/번역/교체 사이클*이 콘텐츠 운영 과제로 추가됨.
- 향후 *온보딩 수집 정보*(나이/건강/일정)도 LLM 프롬프트에 들어가는 경우 *동일한 버킷팅 규칙*을 적용 — 별도 ADR 예정.
- 1일 캡 8회는 튜닝 변수. 사용자별 *발동 빈도 분포*가 데이터로 잡히면 조정.
