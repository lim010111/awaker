# 판별 엔진 학습 라벨(y)은 ESM-보정 운동학 distillation으로 만들고 순간-EMA로 타당성을 측정한다

[[detection-engine]] 이 출력하는 *무지성 스크롤 신뢰도*를 지도학습하려면 윈도우별 정답 y가 필요한데, 우리는 y를 **베타 단계 AccessibilityService 스크롤 운동학(fling cadence·콘텐츠 체류시간·앱맥락)을 룰로 변환한 dense 라벨**로 만든다. 즉 v1 모델은 *권한-경량 센서(X)로, ESM-집계로 보정된 스크롤 운동학 룰(teacher)을 근사하는 distillation*이다 — 이 사실을 미화하지 않는다. y가 *느끼는* 무지성(구성개념) 그 자체를 담보하지 못한다는 한계를 인정하고, **순간-EMA(event-contingent momentary self-report) 프로브로 그 한계의 크기를 *가정하지 않고 측정*한다.** 추론 입력 X는 ADR-0004의 권한-경량 센서 융합으로 고정(터치 0)하고, 라벨 y의 teacher와 검증 라벨만 베타 한정으로 얻는다 — **터치/스크롤은 입력 피처가 아니라 GT 전용**.

## 왜 이 결정인가 — 리서치 근거 (2026-06, deep-research 적대적 검증)

- **무지성 스크롤은 실재하는 학술 구성개념**(goalless·reduced-awareness·dissociative — JCMC 2024 / Binge Scrolling Scale 2025). 단 **검증된 정의는 전부 자기보고 척도이고 운동학적 조작화는 문헌에 없음** → 우리의 운동학 y는 차용이 아니라 새로 만드는 프록시.
- **자기보고 ↔ 행동로그 괴리**: 메타 pooled r≈.38 (Parry 2021), 순간단위(within-person) r≈.30(노이즈 큼), 집계단위(between-person) r≈.52~.55(수용 가능).
- **IMU(자이로/가속도)만으로 폰 상태/스크롤 탐지 가능** — 8개 상태 ~78.6%(Sensors 2020), 거친 상태 분류 천장 ~79%. AccessibilityService 불필요가 기술적으로 성립.
- **reading vs 무지성 스크롤 운동학 분리는 *열린 위험*** — "절대 불가" 강한 주장은 검증에서 기각됐으나, 한 IMU 연구가 둘을 운동 시그니처 유사로 *병합*. 깨끗한 분리 보장 없음.
- **개인화 +0.07 AUC**(Wampfler CHI 2022) — modest, 코어 아님.
- **"도파민 루프"는 실재 기전이나 '스마트폰 중독'은 임상 진단 아님**(DSM-5/ICD-11 제외) → "중독/마음 상태 탐지"를 약속하지 않고 행동 시그니처만 주장. active-passive use 프레임은 타당도 약해(141편 메타) 매핑 대상서 제외.

## 라벨 파이프라인

1. **학습 라벨 (dense, teacher)** — 베타 AS 스크롤 운동학 → 룰 변환 → 윈도우별 y 자동생성. 소형 모델을 채울 유일한 밀도 라벨원. *이 룰이 teacher, 자이로 모델이 student.*
2. **룰 *집계* 보정** — 간헐 ESM으로 *between-person 수준*에서 "운동학 룰이 체감 무지성과 상관되는가"를 확인하고 룰 파라미터를 보정. ⚠️ **이것은 모집단 수준 보정이지, 개별 윈도우 라벨이 옳다는 증거가 아니다**(생태학적 오류 회피). ESM 라벨은 윈도우 학습신호에 직접 투입하지 않는다.
3. **순간-EMA *타당성 프로브* (per-window 측정 전용)** — 예비 임계 진입 시 원탭("지금 직전, 무지성이었나요?"):
   - *규칙-양성* 윈도우 샘플 → AS 룰의 진짜 **per-window precision 측정**.
   - *무작위* 윈도우 샘플 → **recall/false-negative 추정** — 출시 후엔 구조적으로 불가능한 그 수치를 베타에서 확보.
   - 1차 목적은 **distillation 천장을 *측정*하는 것**(가정 아님). 보정(semi-supervised)으로 천장을 *돌파*하는 건 v1 범위 밖 — 측정 결과 보고 별도 결정.
4. **추론 입력 X (베타·출시 동일)** — 자이로/가속도/조도/화면·배터리·시간대/UsageStatsManager. 터치/스크롤 0. 단 "동일 스키마 ≠ 동일 분포"임을 인지(아래 Consequences).
5. **실세계 운영 신호** — [[north-star]](1분 이탈, ADR-0007) + G2(오탐 신고, ADR-0007). **precision만 교정**하는 비대칭 신호임을 명시(false-negative는 출시 후 불가시).

## Considered Options

- **y = 자기보고 직접 라벨 (retrospective ESM/EMA)** — 구성개념 타당성 최고지만 순간단위 r≈.30 + 라벨 희소로 dense 학습 불가. 검증 용도로 강등.
- **y = 순수 스크롤 휴리스틱 distillation (검증 없음)** — 단순하나 천장 = 휴리스틱 품질이고 *천장을 영영 모름*. 비채택.
- **y = 순간-EMA를 dense 라벨로 (천장 돌파 시도)** — 진짜 per-window GT지만 ① 반응성("무지성이냐"고 물으면 무지성 상태가 깨짐 — 제품 본질과 같은 역설) ② 방해 예산 ③ 희소. dense 라벨로는 부적합. **샘플 프로브로만** 채택(파이프라인 #3).
- **y = ESM-보정 운동학 distillation + 순간-EMA 측정 프로브 (채택)** — dense 라벨 확보 + 정직한 한계 + *천장을 측정*하는 균형. semi-supervised 보정은 측정 후로 보류.

## Consequences

- **distillation 천장을 정직하게 인정** — v1 모델의 타당성 상한 = min(룰 품질, 자이로→AS운동학 복원율). "타당성을 회수했다"고 주장하지 않음. 제품 자산의 해자는 *초인적 구성개념 타당성*이 아니라 *권한-경량 추론* 자체. ESM은 룰의 모집단 상관을 올릴 뿐, 윈도우 라벨이나 모델 달성 타당성을 올리지 않음.
- **reading 오염은 추론이 아니라 *라벨 소스*에 있음** — 운동학이 reading/무지성을 못 가르면 룰이 원천에서 오라벨 → 모델이 *체계적 라벨 노이즈*를 학습. 집계 ESM은 이 per-window 편향을 못 잡음. 완화: ESM/EMA 검증을 *앱·콘텐츠 유형별 층화*, reading-우세 앱은 *고불확실 라벨 구역*으로 취급 + dwell/맥락 피처 가중. 보수적 임계(오탐↓ 양성누락↑, ADR-0004 일관).
- **운영 플라이휠은 precision만 교정, recall은 출시 후 불가시** — G2는 false-positive만, 북극성은 체크포인트 발동 전제. 놓친 세션은 무신호. → **recall은 베타 순간-EMA(무작위 프로브)로만 추정**하고, 출시 후엔 측정 불가임을 수용. (필요 시 출시 후 *옵트인 순간 프로브*나 holdout로 재추정 — v2 검토.)
- **train/serve 누수 2종 관리**: (a) *시간정렬 누수* — y 윈도우 경계가 AS 타임스탬프 기반이면 serve 시 재현 불가. → 윈도우 분절은 *X만으로 계산 가능*해야 하거나, 불가 시 누수를 문서화하고 모델 민감도를 측정. (b) *코호트 선택편향* — X·임계가 AS 동의 베타 코호트에 튜닝됨. 출시 모집단엔 AS 라벨이 없어 드리프트 감지 불가 → *비지도 피처 분포 드리프트 모니터링*으로 보완.
- **kill 기준(시작점, 튜닝 가능)**: 집계 ESM↔룰 상관 <.45 → 프록시 기각 / .45~.55 → 룰 재보정 / >.55 → 진행. 순간-EMA per-window precision이 바닥(예 <0.6) → teacher 재설계. 무작위 프로브 recall이 과도히 낮으면 임계/피처 재검토. *검증 단계에 통과/실패 바를 명시*해 "장식적 검증" 방지.
- **전역 모델 v1, 개인화는 후속**(+0.07 AUC는 후속 레이어). **베타 AS 동의 흐름 필수**(목적·기간·삭제 고지, 출시 빌드 AS 경로 제거 — ADR-0004).
- **남은 미결**: GT 자동라벨 *룰 임계/수식*, 윈도우 입자 단위, semi-supervised 보정 여부, 전역 vs 개인화 — 후속 grill/ADR.
