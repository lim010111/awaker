# 세션-스코프 상태는 활성 세션을 따른다 — 후보→후보 전환 포함 단일 리셋 경계

실기기 1일차 검증(이슈 10)에서 세션-스코프 상태 세 가지가 서로 다른 경계를
타고 있음이 드러났다: teacher 룰 fling 윈도우와 체크포인트 시트·가림 카운터는
"활성 세션 없음"(포그라운드가 비후보)에서 리셋되지만 후보→후보 직행 전환에는
반응하지 않고, 체크포인트·N1 기록은 "마지막으로 Started된 세션"에 붙어
Resumed를 무시했다. 결과: 앱 전환을 가로지른 fling 누적, 체크포인트 세션
오귀속(실측 `awaker-82d56acf…` — sbrowser 플링으로 발동한 시트가 YouTube
세션에 기록), 시트의 화면 잔류, 그리고 세션 파일 단위 replay 재현 실패
([[0011-proto-live-trigger-and-offline-replay]] AC 위반 — 1일차 대조 3건 중
2건 불일치).

**활성 세션(현재 포그라운드 후보 앱의 세션)의 변경을 세션-스코프 상태의 단일
리셋 경계로 삼는다.** 폴링 tick이 관측한 활성 세션이 직전과 다르면 — 다른
후보 앱으로의 직행 전환이든 비후보로의 이탈이든 — teacher 룰 fling 윈도우를
비우고, 표시 중인 체크포인트 시트를 내리고, 가림(Extension ordinal) 카운터를
리셋하고, 체크포인트·N1 기록의 겨냥을 새 활성 세션으로 옮긴다. 한 문장으로:
*세션-스코프 상태는 활성 세션을 따라간다.*

**이 방향인 이유: replay가 처음부터 이 의미론이었다.** replay 하네스는 세션
파일의 앱이 포그라운드가 아닌 동안 룰을 리셋한다(away 단절). 기기도
후보→비후보 경계까지는 동일했고, 남은 격차는 후보→후보 직행 전환 하나였다.
즉 이 결정은 새 의미론의 도입이 아니라 **기기를 이미 확정된 replay 의미론에
맞추는 것**이다. 기기와 replay가 같은 5초 폴 포그라운드 스트림을 소비하므로,
리셋 경계·스크롤 라우팅(활성 세션 파일)·로그 파일 경계가 같은 tick에서
일치한다 — 세션 파일 하나만으로 판별이 재현되는 구조가 성립한다.

**트레이드오프: 앱을 오가며 이어지는 무지성 에피소드를 룰이 하나로 보지
못한다.** YouTube 4분 → Chrome 4분 같은 연속 에피소드가 앱 경계에서 끊긴다.
v0의 목적은 GT 라벨 순수성 — "이 fling 윈도우는 어느 앱의 것인가"가 명확해야
student 학습 데이터가 된다 — 이므로 순수성이 연속성보다 값지다. 앱 부류별
임계 분화([[0013-candidate-app-expansion-path]]의 보류 방향)와도 정합:
앱 경계가 곧 룰 상태 경계다.

**예외: 북극성 N1 판정은 리셋을 가로지른다.** N1의 정의는 "표시 후 1분 이내
*후보 앱* 이탈"([[0007-v1-north-star-and-guardrails]]) — 후보→후보 전환은
이탈이 아니므로, 진행 중인 판정은 활성 세션이 바뀌어도 계속된다.

## Considered Options

- **scroll/rule 라인을 열린 모든 세션 파일에 브로드캐스트** — replay가 전량을
  보게 해 재현성만 복구. 기록 중복·용량 증가에 replay pkg 필터 신설이 필요
  하고, 무엇보다 "전역 룰"(앱 경계를 넘는 fling 누적) 의미를 보존하는데 그
  의미 자체가 오귀속의 원인이었다. 비채택.
- **기기-일 단위 단일 로그로 재설계** — 세션을 마커로 강등. 개념적으로 가장
  깨끗하나 파일 수명·export 단위·스키마 전면 변경이라 v0 프로토타입에 과하다.
  비채택 (규모가 커지면 재검토 여지).

## Consequences

- 이 결정 이후 수집되는 GT 라벨은 단일 앱 순수 — 이전(1일차) 로그와 혼합
  분석 시 fling 윈도우 의미가 다름을 전제해야 한다.
- 5초 폴 지연은 남는다: 전환 직후 최대 한 폴 주기의 스크롤이 이전 세션에
  귀속될 수 있다. 기기와 replay가 같은 지연을 겪으므로 재현성은 유지된다.
- 후보→후보 전환 시 시트가 선택 없이 내려간다 — N1 판정은 그대로 진행되므로
  북극성 분모(표시 수)는 불변.
- 세션-스코프 상태의 실효 수명은 "활성 연속 구간"이다: away 유예로 세션이
  이어져도(Resumed) 상태는 새로 시작한다. 이는 기존 동작의 유지이며,
  CONTEXT.md [[세션]]의 "세션 단위 리셋"은 수명의 상한으로 읽는다.
