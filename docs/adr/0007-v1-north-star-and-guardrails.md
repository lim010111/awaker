# v1 북극성 지표 — 체크포인트 1분 이탈률 + 2개 가드레일

v1이 검증해야 할 단 하나의 가설은 *"체크포인트가 그 순간 사용자를 멈추게 하는가"* 다. 따라서 북극성은 *그 순간의 즉시 효능*을 측정하는 지표여야 한다 — 4주 시점의 사용 시간 감소(N3) 같은 지표는 진짜 가치 있지만 *의사결정 사이클이 너무 김*. v1 의사결정 동력으로는 N1을 쓴다.

## 북극성 (North Star)

**체크포인트 표시 후 *1분 이내* 후보 앱 이탈률**

- 분자: 체크포인트 표시 시점 t0 기준 t0+60초 이내에 사용자가 후보 앱을 떠난 체크포인트 수.
- 분모: *같은 세션 중*에 표시된 체크포인트 (세션 종료 후 다른 세션 카운트 안 함).
- **v1 목표: 50% 이상.**
- 측정원: UsageStatsManager 즉시 계산 가능.

## 가드레일

**G1. D7 잔존률 — target 30%+**
북극성이 좋아도 *사용자가 일주일 안에 앱을 지우면* 모든 게 무의미. "체크포인트가 너무 강해서 앱 자체를 삭제하는 가짜 성공" 시나리오 차단.

**G2. 오탐 신고율 — target <10% of 체크포인트**
사용자가 체크포인트에서 "이건 무지성 아냐" 신고하는 비율. [[detection-engine]] 정확도의 직접 신호. 이게 높으면 ADR-0004의 센서 융합 모델/임계 튜닝 필요.

## 장기 (v1.x+)

**N3. 주간 후보 앱 누적 시간 감소율** — 가입 후 1주차 vs 4주차 코호트 비교. v1 출시 후 자연 누적. 진짜 행동 변화의 척도지만 *v1 의사결정엔 너무 늦음*.

## Considered Options

- **세션당 평균 Extension 횟수 (N2)** — Progressive Occlusion의 효과는 보지만 *제품 가치 자체*는 안 봄. 또한 Extension 0회가 "잘 멈췄음"인지 "체크포인트 안 떴음"인지 모호. 비채택.
- **30초 윈도우** — 더 정밀한 즉시성 측정. 그러나 *사용자가 메시지 *읽을* 시간*을 고려하면 너무 짧음. 1분이 균형점.
- **5분/10분 윈도우** — *자연 이탈*(다른 알림, 화장실 등) 노이즈가 섞임.

## Consequences

- 1분, 50%, 30%, 10% 모두 *튜닝 변수*. v1 출시 후 분포 보고 조정. 본 ADR은 *측정 *프레임* 의 고정*이지 숫자의 고정이 아님.
- 데이터 인프라는 *체크포인트 표시 → 후보 앱 이탈 시각*까지를 *모두 온디바이스*에서 측정 + *집계만 익명 전송* 가능해야 함 ([[ADR-0005]] 정신 일관).
- N1 50%를 *데이터로* 달성하면 → N3 측정 시작 시점에서 *진짜 행동 변화 가설*로 무게 이동 가능.
- 가드레일 침범 시 (G1<30% 또는 G2>10%) 북극성 좋아도 *코어 메커니즘 재검토*가 트리거 — 의사결정 룰을 ADR에 박아 *해석의 흔들림* 방지.
