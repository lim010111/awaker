# Awaker

스마트폰에서 **무지성 스크롤** 상태에 빠진 사용자를 *통제*가 아닌 *메타인지 유도*로 깨우는 모바일 서비스. 핵심 자산은 권한-경량 행동 신호(자이로/가속도 중심) 기반의 온디바이스 판별 모델.

## Language

**무지성 스크롤 (Mindless Scroll)**:
의식적 목적 없이 반사적으로 피드를 스크롤하는 *행동 상태*. Awaker가 탐지하려는 단 하나의 상태.
_Avoid_: 숏폼 시청 (영상 포맷명일 뿐, 행동 상태가 아님), 도파민 루프 (왜 위험한가의 *원인 개념*이지 관측 대상이 아님).

**후보 앱 (Candidate App)**:
포그라운드에 떴을 때 판별 엔진을 깨우는 게이트 역할의 앱. 기본 셋(YouTube, Instagram, TikTok, X, Threads 등 피드 앱)에 더해 *기기에 설치된 브라우저가 자동 포함*된다. 사용자가 직접 추가/제외 가능하며, 사용자 제외가 자동 포함보다 항상 우선. 자세한 내용은 ADR-0013.

**판별 엔진 (Detection Engine)**:
후보 앱이 포그라운드인 동안 [[sensor-fusion]] 입력을 받아 *무지성 스크롤 신뢰도*를 출력하는 온디바이스 ML 모델. <1MB, <50ms 추론 목표. 입력은 ADR-0004에 따라 *권한-경량 센서 융합* — AccessibilityService를 쓰지 않음. *프로토타입/베타 빌드에선 student 모델이 아직 없어 AS teacher 룰이 라이브 트리거를 대행* — ADR-0011.

**센서 융합 (Sensor Fusion)**:
판별 엔진의 입력 패키지. 자이로/가속도(3축, 50–100Hz) + 조도 + 화면 상태/배터리/시간대 + 후보 앱 포그라운드 이벤트(UsageStatsManager). *스크롤 이벤트는 직접 받지 않고 자이로 미세 흔들림으로 간접 추론* — 이게 본 제품의 진짜 AI 자산. 자세한 내용은 ADR-0004.

**신뢰도 × 체류 (Confidence × Dwell)**:
체크포인트 발동 룰. 단발성 판정이 아니라 *신뢰도 임계치를 일정 시간 이상 유지*해야 발동. 오탐 억제 + 배터리 절약 + [[0001-checkpoint-message-prefetch]] 의 compute window 역할.

**예비 임계 / 최종 임계 (Pre-emit / Emit Threshold)**:
이중 임계 구조. *예비 임계*(신뢰도 ~0.4) 진입 시 백그라운드 LLM prefetch 시작, *최종 임계*(신뢰도 ~0.7 + dwell 90s) 통과 시 체크포인트 표시. 자세한 내용은 ADR-0001.

**메시지 슬롯 (Message Slot)**:
LLM이 *톤/문장 구조*만 생성하고, *실측 수치*(누적 분, 자세 각도, 세션 N번째 연장)는 시스템이 표시 직전 주입하는 변수 자리. LLM 환각으로부터 사실 수치를 분리하는 장치.

**버킷팅 (Bucketing)**:
LLM이 받는 컨텍스트의 입자 단위. raw 수치 대신 *구간 라벨* 로 추상화 (예: `"01:23"` → `"새벽 1시~2시"`, `"38분 12초"` → `"30~60분"`). 프라이버시 보호와 *문장의 질감* 사이의 절충 — 추상 토큰보다 풍부, raw보다 안전. 자세한 내용은 ADR-0005.

**로컬 템플릿 풀 (Local Template Pool)**:
출시 빌드에 박힌 50~200개의 메시지 톤 풀. LLM 실패/오프라인/일일 비용 캡 초과 시 *silent fallback*. 시스템은 *절대 침묵하지 않음*. 사실 수치는 [[message-slot]] 방식으로 주입.

**LLM 게이트웨이 (LLM Gateway)**:
클라우드 LLM 호출의 단일 진입점. v1은 **OpenRouter** 채택 (ADR-0008) — provider별 직결 대신 라우팅·fallback·통합 빌링을 한 곳에. underlying provider 변경은 *코드가 아닌 config*. 데이터 경로는 [[ADR-0005]] 참조.

**체크포인트 (Checkpoint)**:
판별 엔진이 무지성 스크롤을 확신했을 때 사용자에게 띄우는 *자각 유도 프롬프트*. 멈춤을 강요하지 않음 — 사실 인용 메시지 + 두 가지 응답 선택지로 구성. 강제 잠금/페널티는 의도적으로 배제.
_Avoid_: 개입 (Intervention) — *끊는다*는 뉘앙스라 본 제품 철학과 불일치.

**체류 연장 (Extension)**:
체크포인트의 한 선택지. 사용자가 "1분 더 보겠다"를 의식적으로 선택. 페널티 없음. 메타인지 핵심 레버 — 같은 세션에서 몇 번째 연장인지 / 누적 분이 다음 체크포인트 메시지의 [[message-slot]]으로 노출됨. *연장 단위 1분은 튜닝 가능*.

**자발 종료 (Voluntary Exit)**:
체크포인트의 다른 선택지. 사용자가 폰을 face-down으로 뒤집으면 자이로가 완료를 검증하고 [[refresh-sound]]를 재생. 검증 성공 시 [[progressive-occlusion]] 점유율 즉시 리셋.

**환기 사운드 (Refresh Sound)**:
[[Voluntary Exit]] 검증 후 재생되는 사운드. *번들 풀*(백색/핑크 노이즈 + 빗소리/파도/모닥불 등 5~10개, CC0/MIT) + *사용자 MP3*(Android SAF `ACTION_OPEN_DOCUMENT`로 단일 파일 명시 선택 — 광범위 저장소 권한 회피). 종료 = *사용자 지정 타이머*(기본 15분, 5/15/30/60 선택) 또는 *face-up이 30초 이상 유지*, 둘 중 먼저. *수면 모드*는 사용자 토글 — 활성 시 심야 시간대엔 face-up 정지 비활성화 + 기본 60분.

**점진적 가림 (Progressive Occlusion)**:
체크포인트 바텀 시트의 *높이*가 같은 세션 누적 [[Extension]] 횟수의 함수. 1회차 25%부터 시작해 시각 점유율이 자라며 최대 70%까지. 두 선택지는 항상 살아 있음 — 사라지는 건 *화면의 여백*이지 *선택의 자유*가 아님. 자세한 내용은 ADR-0003.

**세션 (Session)**:
같은 후보 앱에서의 연속 사용 구간. 후보 앱이 *5분 이상* 포그라운드를 떠나거나 [[Voluntary Exit]]이 성공 검증되면 세션 종료. 누적 Extension 카운터와 점진적 가림 점유율은 *세션 단위*로 리셋. [[north-star]] 분모도 세션 단위로 계산.

**활성 세션 (Active Session)**:
현재 포그라운드에 있는 후보 앱의 [[세션]]. 여러 세션이 away 유예로 동시에 열려 있어도 활성 세션은 항상 최대 하나 (포그라운드가 비후보면 없음). 판별 상태·체크포인트·[[progressive-occlusion]] 카운터 등 *세션-스코프 상태는 전부 활성 세션에 귀속*되며, 활성 세션이 바뀌면 — 다른 후보 앱으로의 직행 전환 포함 — 함께 리셋·재귀속된다. 예외는 [[north-star]] 판정 하나 (후보 앱 *전체* 이탈 기준이라 전환을 가로질러 진행). 자세한 내용은 ADR-0014.

**북극성 (North Star)**:
v1 의사결정의 단일 기준 지표 — *체크포인트 표시 후 1분 이내 후보 앱 이탈률*. 목표 50%+. 가드레일: D7 잔존률 30%+, 오탐 신고율 <10%. 자세한 내용은 ADR-0007.

## Relationships

- **후보 앱** 포그라운드 → **판별 엔진** wake up.
- **판별 엔진** → [[sensor-fusion]] 입력 → 무지성 스크롤 신뢰도. (터치/스크롤은 *입력 아님* — 베타 GT 전용; 입력 X는 ADR-0004, 라벨 y는 ADR-0010)
- **신뢰도 × 체류** 임계 통과 → **체크포인트** 표시.
- **체크포인트** → 사용자가 **체류 연장** 또는 **자발 종료** 중 선택.
- **자발 종료** 선택 → 자이로 face-down 감지 → 환기 사운드 재생.

## Example dialogue

> **Dev**: "사용자가 유튜브로 20분짜리 강의를 본다. 판별 엔진 발동?"
> **도메인**: "YouTube는 후보 앱이라 엔진은 깨어남. 그런데 강의는 한 영상에 오래 머무름 — 반복적 스크롤 관성이 없음 — 신뢰도가 낮게 유지됨. 개입 없음."

> **Dev**: "점심시간에 X 피드를 2분 스크롤한다. 무지성 스크롤?"
> **도메인**: "패턴에 달림. 뭔가 찾으려고 (속도 변화 있고 멈춰서 읽는 구간 있고) 스크롤하면 무지성 아님. 2초 간격으로 반사적으로 넘기고 멈춤이 없으면 무지성. 판별 엔진이 이걸 가려야 함 — 현재 가설 단계, 논문 검토 필요."

**Progressive Onboarding**:
3단계 점진 정보 수집 — *0일차 미니멈* (목표 + 권한) → *3~7일차 보통* (나이대/직업/시간대) → *2주차+ 깊이* (Health Connect/만성질환/수면). 각 항목 거부 가능, 거부 시 LLM 프롬프트는 그 슬롯 단순 생략. 자세한 내용은 ADR-0006.

## Flagged ambiguities

- **무지성 스크롤 vs 일반 스크롤 경계** (리서치 2026-06 완료 / y 정의 ADR-0010 확정): 무지성 스크롤은 *실재하는 학술 구성개념*(JCMC 2024 / Binge Scrolling Scale 2025)이나 검증된 정의는 전부 *자기보고 척도* — 운동학적 조작화는 문헌에 없어 우리가 새로 만드는 프록시. ADR-0010 결정: y = *ESM-보정 운동학 distillation + 순간-EMA 타당성 프로브*(천장을 가정 않고 측정). 잔존 위험 ①: reading↔무지성 운동학 분리가 *라벨 소스*를 오염시킬 수 있음(보수적 임계 + 콘텐츠 체류시간/앱맥락으로 완화). ②: 출시 후 recall이 구조적으로 불가시 — 베타 무작위 프로브로만 추정. 남은 미결: GT 룰 임계/수식, 윈도우 입자 단위.
- **무지성 스크롤에 영상 포맷 포함 범위**: 현재 정의는 영상(Shorts/Reels/TikTok)과 텍스트 피드(X/Threads/IG 피드)를 모두 포함. 음악 스트리밍/뉴스 인피니트 스크롤 등 추가 매체는 미정.
